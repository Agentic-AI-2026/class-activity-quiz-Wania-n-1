Quiz: Convert Planner–Executor Agent to LangGraph
Objective

You are given a working Planner–Executor agent implemented in LangChain.
Your task is to convert this agent into a LangGraph workflow while preserving the same functionality.

Provided
Existing Planner–Executor agent code (LangChain)
Tool implementations (already working)
Requirements
1. Define State
Create a state structure that represents the agent’s data.
Your state should include:
goal → user input
plan → list of steps generated by planner
current_step → step tracker
results → outputs from executed steps
2. Planner Node
Implement a node that:
Takes the user goal
Calls the LLM
Generates a structured plan
Stores the plan in the state
3. Executor Node
Implement a node that:
Executes one step at a time
If a step contains a tool, call the appropriate tool
Otherwise, use the LLM for synthesis
Store the result
Move to the next step
4. Graph Flow
Construct a LangGraph workflow with the following behavior:
START → planner_node → executor_node → (loop until complete) → END
The graph must execute steps sequentially and stop when all steps are completed
5. Tool Handling
Ensure your implementation correctly:
Identifies the tool to call
Passes correct arguments
Handles cases where no tool is required
Test Case

Your implementation should work for inputs like:

goal = "Fetch Q3 sales data and summarize it."
Constraints
Do not hardcode outputs
Do not skip the planning phase
Each iteration should execute only one step
Maintain the same logical flow as the original agent

## ✅ Implementation Complete

### Solution Overview

This solution successfully converts the Planner-Executor agent from LangChain to LangGraph while maintaining all core functionality with **working MCP tools**.

### Architecture

**State Definition** (`PlannerExecutorState` in `graph.py`):
```python
- goal: str              # User input
- plan: list[dict]      # Steps generated by planner
- current_step: int     # Step tracker
- results: list[dict]   # Outputs from executed steps
- completed: bool       # Workflow completion flag
```

**Graph Flow**:
```
START → Planner Node → Executor Node ⟲ (loop) → END
```

### Components

#### 1. **Planner Node** (`graph.py`)
- Takes the user goal
- Calls LLM to generate ordered JSON plan
- Validates and stores plan in state
- Each step includes: step number, description, tool name (or null), and arguments

#### 2. **Executor Node** (`graph.py`)
- Executes one step per iteration
- Checks if step requires a tool:
  - **Tool steps**: Calls the appropriate tool with correct arguments
  - **Synthesis steps**: Uses LLM with context from prior results
- Handles both synchronous and asynchronous tool invocations
- Stores results and advances to next step
- Returns `completed=True` when all steps finish

#### 3. **Router/Conditional Edge** (`graph.py`)
- `should_continue()` function routes based on state
- Continues loop if more steps remain
- Transitions to END when all steps complete

### Setup & Configuration

**1. Install Dependencies**
```bash
pip install langgraph langchain langchain-groq nest-asyncio python-dotenv mcp tavily-python
```

**2. Configure LLM Provider** (`.env`)

Choose one provider:

**Option A: Groq** (llama-3.1-8b-instant - Fast & Free) ⭐ **RECOMMENDED**
```
LLM_PROVIDER=groq
GROQ_API_KEY=your_groq_api_key
```

**Option B: Anthropic Claude**
```
LLM_PROVIDER=anthropic
ANTHROPIC_API_KEY=sk-ant-...
```

**Option C: Google Gemini**

### Tool Usage

When the agent encounters a tool in the plan:
1. **Identifies** the tool name (e.g., `fetch_data_source`)
2. **Extracts** arguments from the plan (e.g., `source: "sales"`)
3. **Calls** the tool via MCP with `ainvoke()` wrapper
4. **Stores** the result in state for next steps
5. **Falls back** to LLM synthesis if tool unavailable
